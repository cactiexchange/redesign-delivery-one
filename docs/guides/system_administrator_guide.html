2 · NetBox: the source of truth & data model

Every deployment is a NetBox tenant. The deploy tool reads intent from NetBox and writes outcomes back. Nothing about a project lives only in a spreadsheet or someone's head.

What the seeder creates

ObjectNotes
TenantOne per project (slug = project name). Carries the storage_architecture custom field.
Device rolesServer, Storage, Network, and Customer Switch (grey — visually distinct gear we don't configure).
ManufacturersDell, plus Customer-Provided for customer switches.
Device typesFrom hardware_definitions.json; u_height + a facilities summary comment.
DevicesOne per node, status planned, with all deployment intent in local_context_data.

Key fields

  • storage_architecture — vsan_primary | array_primary | array_only. Branches config synthesis and the DAG sanity checks. Stored on the tenant and stamped into each device's context so the offline pull is self-contained.
  • customer_provided (bool) — marks switches the tool advises on but never configures.
  • Per-product intent (IPs, VLANs, VLT domains, BGP ASNs…) lives in local_context_data, resolved per node by the seeder (.10/.11/…, VLT-peer, ASN increment).
Device types are populated from the hardware matrix. Run python scripts/build_custom_device_types.py (DevOps role) to sync NetBox device types after a matrix change. It derives u_height from the facilities data and retires stale types.

3 · Security model

The field binary is built to be safe to hand an L2 engineer and to survive an air-gapped site. Three layers:

License & identity (per-engineer build)

  • Each engineer gets a uniquely compiled exe (cmd/admin/admin_builder.go) with their name, an expiry date, and a random BuildSecret injected via -ldflags and recorded in the NOC registry (minted_builds.json, never committed).
  • Offline unlock uses HMAC-SHA256 challenge-response — the engineer reads a Challenge Code, the NOC computes the override (admin_builder respond). There is no master keyword; the old admin-override and ×42 arithmetic were removed.
  • Anti-rollback: a MAC-protected license_state.json floors the trusted clock so a rolled-back workstation clock can't revive an expired build.

Encryption

  • The hardware matrix ships as hardware_matrix.enc (AES-GCM, build-time master key).
  • The credential vault (credentials_vault_<tenant>.json.enc) and the session bundle (session_bundle_<tenant>.enc) are AES-GCM under a key derived from BuildSecret + engineer — so they only open in the binary that wrote them, and survive process restarts on site.

NetBox tokens (no compiled-in fallback)

The legacy shared token was removed from the binary (finding RT-2). Token resolution is NETBOX_TOKEN env → mint-time injected token → hard error. Scope per phase:

PhaseInternetToken scope
check-in / --onlineyesread on dcim, tenancy, extras
deploy (default)nonone — runs from the bundle
check-outyeswrite on dcim.device + extras.journalentry
Outstanding RT-2 action (server-side): the legacy token (K9pv…) is gone from the Go binary but must still be revoked in NetBox, and the Python admin scripts still carry it as a default — rotate those. Tracked in the deploy repo's red-team doc.
TLS to factory devices: InsecureSkipVerify / InsecureIgnoreHostKey are scoped to factory-bootstrap flows only (self-signed certs / fresh host keys on first contact). Documented trade-off, not a blanket setting.

5 · Engagement lifecycle (system view)

  1. Design — diagrammer wizard or intake workbook.
  2. Seed — into NetBox via the seeder (DevOps role), validated by the DAG.
  3. PEQ — customer requirements doc generated from the tenant.
  4. Mint — a per-engineer exe + sealed hardware matrix (DevOps role).
  5. Check-in — engineer seals the project to a bundle (online).
  6. Deploy — air-gapped, from the bundle.
  7. Check-out — status + journals back to NetBox; as-built package out; optional 1Password flush.
  8. As-built — reparsers-v2 turns the package (+ TSRs) into the customer document.

7 · System-level troubleshooting

SymptomSystem cause / action
Engineer's exe fatally locksBuild expired or clock anti-rollback tripped. Re-mint (DevOps); never patch the clock.
Every cold start shows a Challenge Codetime.cloudflare.com unreachable on that network; license falls back to NOC override. Expected — supply the override, or pre-warm license_state.json online first.
check-out 403Token lacks write scope (dcim.device / extras.journalentry). Issue a write-scoped token.
DAG rejects a valid-looking designIntent violates a hardware-matrix rule (NIC, node count, capacity, firmware floor). Fix the design; do not bypass.
NetBox device types look wrong / missing modelsRe-run build_custom_device_types.py after a matrix change (DevOps).
A bundle won't open on a different laptopBy design — bundles are keyed to the minting binary. Re-check-in with that engineer's build.
